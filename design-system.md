# 网站设计系统规范

## 1. 设计理念

我们的设计系统基于专业、现代和用户体验优先的原则，旨在创建一个视觉吸引力强、交互流畅且易于维护的网站。设计系统采用了分层设计方法，确保了视觉层次分明，信息传递清晰。

## 2. 色彩系统

- **主色 (Primary)**: `#2563eb` - 专业蓝色，代表信任、可靠和专业
- **主色亮色 (Primary Light)**: `#3b82f6` - 用于悬停状态和渐变效果
- **主色暗色 (Primary Dark)**: `#1d4ed8` - 用于激活状态和强调效果

- **辅助色暗色 (Secondary Dark)**: `#ea580c` - 用于激活状态

## 3. 排版系统

### 3.1 字体
- **主要字体**: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif
- **次要字体**: 'Playfair Display', serif (用于特殊标题和强调)

### 3.2 字体大小
- **H1**: 3rem (48px) - 主标题
- **H2**: 2.5rem (40px) - 章节标题
- **H3**: 2rem (32px) - 子章节标题
- **H4**: 1.5rem (24px) - 卡片标题
- **正文**: 1rem (16px) - 主要内容
- **小文本**: 0.875rem (14px) - 辅助信息

### 3.3 行高
- **大标题**: 1.25 - 1.3
- **正文**: 1.65 - 1.75
- **小文本**: 1.5

### 3.4 字重
- **Bold**: 700 - 标题和强调文本
- **Medium**: 500 - 正文和按钮
- **Regular**: 400 - 辅助文本

## 4. 间距系统

### 4.1 基础间距
- **XS**: 0.25rem (4px)
- **SM**: 0.5rem (8px)
- **MD**: 1rem (16px)
- **LG**: 1.5rem (24px)
- **XL**: 2rem (32px)
- **2XL**: 3rem (48px)
- **3XL**: 4rem (64px)

### 4.2 应用规则
- 内容块间距: 2XL - 3XL
- 卡片间距: XL - 2XL
- 元素间距: MD - LG
- 内边距: LG - XL

## 5. 边框与圆角

### 5.1 边框宽度
- **细边框**: 1px - 用于分隔线和卡片
- **粗边框**: 2px - 用于按钮和高亮元素

### 5.2 圆角半径
- **SM**: 0.375rem (6px) - 小按钮和输入框
- **MD**: 0.5rem (8px) - 卡片和中等按钮
- **LG**: 0.75rem (12px) - 大按钮和特殊卡片
- **XL**: 1rem (16px) - 英雄区域和大型容器

## 6. 阴影系统

### 6.1 阴影层级
- **SM**: 0 1px 2px 0 rgba(0, 0, 0, 0.05) - 轻微阴影
- **MD**: 0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -1px rgba(0, 0, 0, 0.06) - 卡片和元素
- **LG**: 0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -2px rgba(0, 0, 0, 0.05) - 悬浮状态
- **XL**: 0 20px 25px -5px rgba(0, 0, 0, 0.1), 0 10px 10px -5px rgba(0, 0, 0, 0.04) - 模态框和重要元素

## 7. 过渡与动画

### 7.1 过渡时间
- **Fast**: 0.15s - 快速状态变化
- **Base**: 0.3s - 基础过渡效果
- **Slow**: 0.5s - 平滑动画

### 7.2 过渡函数
- **Ease-in-out**: 用于平滑的状态变化
- **Ease-out**: 用于进入动画
- **Ease-in**: 用于退出动画

### 7.3 动画效果
- **Fade-in**: 元素进入时的淡入效果
- **Slide-in**: 元素从一侧滑入
- **Zoom**: 元素缩放效果
- **Pulse**: 呼吸效果
- **Hover-lift**: 元素悬浮时的轻微上浮

## 8. 组件设计

### 8.1 按钮
- **主按钮**: 蓝色背景，白色文字，中等圆角，悬停时有上浮效果
- **辅助按钮**: 深色背景，白色文字，中等圆角
- **轮廓按钮**: 透明背景，蓝色边框，蓝色文字
- **大按钮**: 更大的尺寸和内边距，用于英雄区域

### 8.2 卡片
- **标准卡片**: 白色背景，细边框，中等圆角，轻微阴影，悬停时有上浮和阴影增强效果
- **特色卡片**: 带有渐变背景或特殊边框的卡片

### 8.3 导航
- **顶部导航**: 白色背景，细边框，阴影，滚动时背景半透明
- **下拉菜单**: 白色背景，细边框，较大阴影，圆角
- **移动端菜单**: 侧边抽屉式菜单，带有渐变标题

### 8.4 页脚
- **深色背景**: 深色背景，白色文字，带有渐变顶部边框
- **链接样式**: 悬停时有箭头指示和颜色变化
- **分隔线**: 细分隔线，带有渐变中心点

## 9. 布局原则

### 9.1 网格系统
- **容器宽度**: 最大1200px
- **响应式断点**: 480px, 768px, 992px, 1200px
- **列数**: 12列网格系统

### 9.2 视觉层次
- **清晰的层级结构**: 使用字体大小、颜色和间距创建清晰的视觉层级
- **呼吸空间**: 充足的留白，提升内容可读性
- **对齐原则**: 元素对齐一致，保持视觉整洁

### 9.3 响应式设计
- **移动端优先**: 从移动端开始设计，逐步扩展到桌面端
- **弹性布局**: 使用弹性盒和网格布局，确保元素在不同屏幕尺寸下正确排列
- **内容适配**: 根据屏幕尺寸调整内容大小和布局结构

## 10. 可访问性

### 10.1 色彩对比度
- 遵循WCAG 2.1 AA级标准，文本与背景对比度至少为4.5:1
- 大文本(18px+)对比度至少为3:1

### 10.2 键盘导航
- 所有交互元素均可通过键盘访问
- 清晰的焦点状态指示

### 10.3 语义化HTML
- 使用正确的HTML标签，确保屏幕阅读器能够正确解读内容
- 为图片添加alt属性

## 12. 开发指南

### 12.1 CSS变量
- 使用CSS变量管理所有设计属性，确保一致性和易于维护
- 变量命名遵循BEM命名规范

### 12.2 组件开发
- 遵循原子设计原则，将UI拆分为原子、分子、组织和模板
- 组件命名清晰，功能单一

### 12.3 代码规范
- 遵循团队代码规范，确保代码质量和一致性
- 使用Prettier和ESLint进行代码格式化和检查

## 13. 维护与更新

### 13.1 版本控制
- 设计系统的变更需经过审核和版本控制
- 每个版本的变更需记录在变更日志中

### 13.2 反馈机制
- 建立反馈机制，收集用户和开发团队的反馈
- 定期更新设计系统，适应业务和技术变化
